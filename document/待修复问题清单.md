# 待修复问题清单

## 1. QwenStrategyBackend 低质量阈值硬编码与运行配置不一致

### 问题级别

中高优先级

### 问题位置

- `backend/app/services/backends.py`
- 类：`QwenStrategyBackend`
- 位置特征：
  - `SYSTEM_PROMPT`
  - `user_message`

### 问题现象

系统在 `config.py` 中已将：

- `low_quality_threshold` 从 `0.35` 调整为 `0.15`

但在开启 `LLM_STRATEGY_ENABLED=1` 时，`QwenStrategyBackend` 仍会将低质量复核条件按：

- `quality_score < 0.35`

进行解释和策略选择，导致大量本不应进入低质量复核的样本被错误分流到：

- `selected_strategy = review_only`

### 实际排查证据

已确认以下事实：

1. 运行环境读取到的配置值为 `0.15`。
2. 任务结果中的 `model_versions` 显示阈值文件读取正常。
3. 但 `strategy_selection.details.thinking_chain` 中仍明确写出：
   - “低于系统设定的 0.35 低质量阈值”
4. 因此问题不在 `config.py` 未生效，而在 LLM 策略提示词仍保留旧规则。

### 问题影响

该问题会导致：

1. LLM 策略层与本地规则配置不一致。
2. 开启 LLM 策略时，批量评测结果不稳定。
3. 外部测试集中大量样本被误判为 `review_only`。
4. 实验结果受提示词旧规则干扰，降低可复现性。
5. 系统展示模式与实验模式的结果差异被放大。

### 当前临时绕行方案

在批量实验模式下：

- 设置 `LLM_STRATEGY_ENABLED=0`

作用：

1. 关闭 `qwen_strategy` 策略选择。
2. 回退到规则式策略选择链路。
3. 保证批量评测与本地配置保持一致。
4. 降低 API 开销并提高实验复现性。

### 正式修复建议

应将 `QwenStrategyBackend` 中的低质量阈值说明改为动态读取配置，而不是写死为 `0.35`。

建议修复方向：

1. 在 `SYSTEM_PROMPT` 中将：
   - `quality_score<0.35`
   替换为基于 `settings.low_quality_threshold` 的动态字符串。

2. 在 `user_message` 中补充当前质量复核阈值说明，例如：
   - `当前低质量复核阈值：quality_score<{settings.low_quality_threshold}`

### 当前处理策略

本项目现阶段采用双模式策略：

1. 系统演示模式
   - `LLM_STRATEGY_ENABLED=1`
   - 用于展示智能体协作、大模型策略选择与推理链输出

2. 批量实验模式
   - `LLM_STRATEGY_ENABLED=0`
   - 用于保证定量实验稳定、可控、可复现

### 后续处理建议

1. 论文初稿阶段不以此问题阻塞实验推进。
2. 在论文主实验完成后，安排一次针对 `QwenStrategyBackend` 的正式修复。
3. 在答辩演示前，建议完成修复并复验单图推理链输出逻辑。
