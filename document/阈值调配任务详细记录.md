# 阈值调配任务详细记录

## 1. 任务背景

本任务针对深度伪造人脸检测系统在外部数据集 Celeb-DF 上出现的大量 `review_required` 样本进行排查和优化。  
前期批量评测结果表明，系统在 200 张平衡测试图片上自动判定能力较弱，大量样本被送入人工复核，导致整体可用性较差，不适合直接作为论文中的最终实验结果。

本轮工作的目标不是立即重训模型，而是先验证：

1. 问题是否主要由阈值设置不合理导致。
2. 在不改动模型权重的前提下，是否可以通过阈值校准降低 `review` 比例。
3. 阈值调整后，系统在 Celeb-DF 外部测试集上的自动判定准确率能够提升到什么水平。

## 2. 本轮输入数据

### 2.1 评测数据来源

- 数据集：Celeb-DF-v2
- 评测方式：从官方测试视频中抽帧，生成带标签图片
- 标签规则：
  - `real__*.jpg` 表示真实样本
  - `fake__*.jpg` 表示伪造样本

### 2.2 本轮阈值校准使用的数据规模

- 总样本数：200
- 已标注样本数：200
- 真实样本数：100
- 伪造样本数：100

### 2.3 阈值校准前的评测摘要

批量评测摘要如下：

- `total_images = 200`
- `correct = 12`
- `incorrect = 188`
- `accuracy = 0.06`
- `review_required = 184`
- `failed_or_timeout = 0`

该结果说明默认阈值下系统过于保守，绝大多数样本未形成明确二分类输出。

## 3. 阈值校准脚本与运行目的

### 3.1 使用脚本

脚本路径：

- `backend/scripts/calibrate_thresholds_offline.py`

该脚本作用为：

1. 读取已有批量评测 CSV。
2. 对 `true_threshold` 与 `fake_threshold` 进行网格扫描。
3. 统计每组阈值下的：
   - 总体准确率
   - 自动覆盖率
   - 人工复核率
   - 真实样本准确率
   - 伪造样本准确率
   - 假阳性与假阴性数量
4. 输出推荐阈值组合。

### 3.2 运行命令

在项目根目录下执行：

```powershell
python backend\scripts\calibrate_thresholds_offline.py `
  --input-csv "D:\datasets\celebdf_balanced_eval.csv" `
  --output-json "D:\datasets\threshold_calibration_results.json" `
  --output-csv "D:\datasets\threshold_calibration_results.csv" `
  --true-min 0.00 `
  --true-max 0.50 `
  --true-step 0.05 `
  --fake-min 0.50 `
  --fake-max 1.00 `
  --fake-step 0.05 `
  --top-k 10 `
  --min-coverage 0.20
```

## 5. 结果解读

### 5.1 直接结论

推荐优先采用：

- `true_threshold = 0.05`
- `fake_threshold = 0.50`

对应表现：

- `accuracy = 0.5850`
- `coverage = 1.0000`
- `review = 0.0000`
- `real_acc = 0.34`
- `fake_acc = 0.83`

### 5.2 含义分析

该结果说明：

1. 默认阈值确实设置过于保守。
2. 经过阈值校准后，系统不再大规模输出 `review_required`。
3. 但总体准确率仅为 58.5%，说明问题不只是阈值，还包括模型对 Celeb-DF 数据域的泛化不足。
4. 系统更倾向于识别伪造样本，对真实样本误报偏高。

### 5.3 结论定位

本轮阈值调配的结论应定义为：

- 解决了“默认阈值导致大量人工复核”的问题。
- 暂未彻底解决“跨数据集泛化不足”的问题。
- 为后续是否继续重训提供了定量依据。

## 6. 后续操作步骤

### 6.1 在系统中暂时应用校准阈值

在实际运行设备上找到融合模型元数据文件或阈值配置位置，将阈值改为：

- `true_threshold = 0.05`
- `fake_threshold = 0.50`

如果系统阈值来自融合模型元数据文件，优先检查：

- `backend/storage/models/xgboost_fusion.meta.json`

如果系统阈值来自配置文件，则检查对应配置项。

### 6.2 进行小规模复验

不要直接跑大规模评测，先做 40 到 60 张平衡样本复验：

- 真实图片：20 到 30 张
- 伪造图片：20 到 30 张

目的：

1. 验证离线校准结论能否在真实 API 流程中复现。
2. 观察是否仍存在大量 `review_required`。
3. 检查真实样本是否仍存在明显误报。

### 6.3 复验建议命令

```powershell
python backend\scripts\batch_evaluate_api.py `
  --input-dir "D:\datasets\celebdf_eval_images" `
  --api-base http://127.0.0.1:8000/api `
  --output-csv "D:\datasets\celebdf_threshold_verify.csv" `
  --output-json "D:\datasets\celebdf_threshold_verify.json" `
  --shuffle `
  --seed 42 `
  --max-real 20 `
  --max-fake 20 `
  --flush-every 1
```

### 6.4 复验后应重点检查

重点检查以下指标：

1. `review_required` 是否接近 0。
2. `accuracy` 是否接近离线校准结果。
3. `real_accuracy` 是否依然偏低。
4. 是否出现全部偏向 `fake` 的现象。

## 7. 论文中如何保留这项工作

该任务不应只写成“调了阈值”，而应保留为一个完整实验步骤：

1. 发现问题：默认阈值下大量样本进入人工复核。
2. 分析原因：先排除阈值设置问题，而不是直接重训。
3. 设计方法：对真假阈值进行离线网格搜索。
4. 形成结果：覆盖率提升到 100%，复核率降到 0，但准确率仅提升到 58.5%。
5. 得出结论：阈值优化能提升可用性，但不能替代模型泛化能力提升。

这能体现你的调试过程是系统性的，而不是只报一个最终数字。

## 9. 当前阶段建议

当前阶段不建议立刻重训。优先顺序应为：

1. 先将校准阈值写入系统并做小规模复验。
2. 复验通过后，将“默认阈值 vs 校准阈值”写入论文实验部分。
3. 如果真实样本误报仍然明显，再决定是否进行进一步重训或域适配。

## 10. 当前任务可直接作为论文支撑的结论

可保留为以下结论：

- 默认阈值配置在外部数据集上导致过高的人工复核率。
- 通过离线阈值扫描可显著降低复核比例，提高系统自动判定覆盖率。
- 阈值调优后系统在 Celeb-DF 上仍存在真实样本误报偏高问题，说明融合模型仍需继续优化跨域泛化能力。
