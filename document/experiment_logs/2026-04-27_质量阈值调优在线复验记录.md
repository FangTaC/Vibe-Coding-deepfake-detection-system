# 2026-04-27 质量阈值调优在线复验记录

## 实验编号

`2026-04-27-quality-threshold-online-verify-01`

## 实验目标

验证 Celeb-DF 外部测试集中大量 `review_required` 是否主要由系统质量门控阈值过高导致，并观察在下调质量阈值后的在线 API 批量评测效果。

## 背景结论

前序排查已确认：

1. 融合模型阈值已实际生效。
2. 当前在线后端读取到的阈值来自 `fusion_metadata`。
3. 实际生效阈值为：
   - `true_threshold = 0.05`
   - `fake_threshold = 0.50`
4. 大量样本进入 `review_required` 的直接原因不是 XGBoost 融合阈值，而是策略阶段命中 `review_only`。
5. 单样本 API 返回的 `review_reason` 为：
   - `图像质量过低或输入不满足系统适用范围。`

## 本轮配置变更

本轮仅调整质量门控阈值：

- 修改前：`low_quality_threshold = 0.35`
- 修改后：`low_quality_threshold = 0.15`

说明：

- 融合阈值保持不变。
- 本轮实验的目的仅为验证质量门控阈值对 `review_required` 的影响。

## 数据集与样本设置

- 数据集：Celeb-DF-v2 抽帧图像
- 输入目录：`D:\datasets\celebdf_eval_images`
- 本轮样本数：40
- 真实样本：20
- 伪造样本：20
- 评测方式：调用现有单图检测 API 逐张评测

## 结果解读

本轮结果表明：

1. 下调质量阈值后，`review_required` 明显下降。
2. 系统已能够对绝大多数样本给出明确二分类结果。
3. 总体准确率达到 `65%`，相较于先前“大量 review、自动判定能力弱”的状态有明显改善。
4. 当前系统仍明显偏向判定为 `fake`，导致真实样本误报偏高。

关键现象：

- `review_required = 2 / 40 = 5%`
- `fake_accuracy = 0.85`
- `real_accuracy = 0.45`
- `false_positive = 10`

## 阶段性结论

本轮实验可以得出以下结论：

1. 在线批量评测中的高复核率，主因是质量门控阈值过高，而不是融合阈值未生效。
2. 在保持 `xgboost` 融合阈值不变的条件下，仅调整 `low_quality_threshold` 即可显著提升系统自动判定覆盖率。
3. 当前系统在 Celeb-DF 外部测试集上的主要剩余问题是：
   - 真实样本误报偏高
   - 系统偏向输出 `fake`

## 可用于论文的实验结论

可直接提炼为：

- 在外部测试集中，系统默认质量门控策略会导致大量样本提前进入人工复核流程。
- 将质量阈值由 `0.35` 下调至 `0.15` 后，系统在线评测的复核率显著下降至 `5%`，总体准确率提升至 `65%`。
- 该结果说明，系统最终检测表现不仅受融合模型影响，也显著受工程化门控策略影响。
