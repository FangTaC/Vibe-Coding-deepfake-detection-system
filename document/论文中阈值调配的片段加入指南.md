# 论文中阈值调配的片段加入指南

## 1. 这一部分在论文中的定位

阈值调配不应写成零散备注，建议作为“实验分析”或“系统优化”中的一个独立小节。  
推荐标题可选：

1. `4.X 阈值校准实验`
2. `4.X 基于外部测试集的决策阈值优化`
3. `4.X 融合决策阈值调优与结果分析`

其作用是回答两个问题：

1. 为什么原系统在外部数据集上会产生大量人工复核结果。
2. 经过阈值优化后，系统可用性改善到了什么程度。

## 2. 建议放入论文的内容结构

建议按下面 5 段来写。

### 2.1 问题提出

写清楚：

- 在 Celeb-DF 外部测试集上进行评测时，系统在默认阈值下出现大量 `review_required` 样本。
- 该现象影响了系统自动判定能力，也不利于后续论文中展示稳定的检测结果。
- 因此，需要在不改动模型结构的前提下，先考察阈值设置对最终判定结果的影响。

### 2.2 方法说明

写清楚：

- 基于已有批量评测输出结果，不重新推理图片，而是离线读取 CSV 结果。
- 对 `true_threshold` 和 `fake_threshold` 做网格搜索。
- 以准确率、覆盖率、人工复核率、真实样本准确率、伪造样本准确率为指标进行比较。

### 2.3 实验设置

建议明确写出：

- 数据集：Celeb-DF-v2 外部测试集抽帧图像
- 样本规模：200 张平衡样本，其中真实和伪造各 100 张
- 参数扫描范围：
  - `true_threshold: 0.00 ~ 0.50`
  - `fake_threshold: 0.50 ~ 1.00`
  - 步长均为 `0.05`

### 2.4 实验结果

应写明最优结果：

- `true_threshold = 0.05`
- `fake_threshold = 0.50`
- `accuracy = 58.5%`
- `coverage = 100%`
- `review_rate = 0%`
- `real_accuracy = 34%`
- `fake_accuracy = 83%`

### 2.5 结果分析

分析建议采用下面逻辑：

1. 默认阈值设置过于保守，导致大量样本无法自动完成二分类判定。
2. 阈值调优后，系统自动覆盖率明显提升，人工复核率显著下降。
3. 尽管可用性提高，但真实样本准确率仍偏低，说明系统对外部数据域的泛化能力不足。
4. 因此，阈值校准解决的是决策边界问题，而不是根本性的跨域识别问题。

## 3. 可直接改写进论文的片段模板

下面这段可以直接作为论文初稿基础文本，再按你的整体语气统一修改。

```text
在系统外部测试阶段，本文选取 Celeb-DF-v2 数据集中的测试视频进行抽帧，并构建了包含 200 张图像的平衡评测集，其中真实样本和伪造样本各 100 张。初始实验发现，系统在默认阈值配置下存在大量样本进入人工复核流程的问题，导致自动判定覆盖率较低，不利于对系统检测性能进行稳定评估。

为分析该问题是否由决策阈值设置不合理所致，本文在不改变模型结构和权重参数的前提下，对融合决策中的真实阈值与伪造阈值进行离线网格搜索。实验以总体准确率、自动判定覆盖率、人工复核率以及真实样本与伪造样本的分类准确率作为评价指标。

实验结果表明，当 true_threshold 取 0.05、fake_threshold 取 0.50 时，系统在该评测集上取得了相对较优的综合表现。此时总体准确率为 58.5%，自动判定覆盖率达到 100%，人工复核率降为 0%，其中真实样本识别准确率为 34%，伪造样本识别准确率为 83%。该结果说明，原始系统默认阈值设置偏保守，经过阈值校准后系统的自动决策能力得到明显改善。

但进一步分析发现，尽管阈值优化显著降低了人工复核比例，系统在真实样本上的误报仍然较高，说明当前模型对 Celeb-DF 外部数据域的泛化能力仍有限。因此，阈值调优可以作为提升系统可用性的有效工程手段，但若要进一步提升跨数据集检测性能，仍需结合模型重训、特征优化或域适配方法继续改进。
```

## 4. 论文中建议配套加入的表格

至少建议加入两张表。

### 4.1 表一：阈值调优前后对比表

建议列：

- 阈值设置
- Accuracy
- Coverage
- Review Rate
- Real Accuracy
- Fake Accuracy

建议行：

- 默认阈值
- 校准阈值 `true=0.05, fake=0.50`

### 4.2 表二：候选阈值组合对比表

从扫描结果中选 5 组左右代表性阈值写入：

- `0.05 / 0.50`
- `0.05 / 0.55`
- `0.10 / 0.50`
- 其他有代表性的组合

目的：

- 证明你不是拍脑袋选阈值
- 说明阈值变化对覆盖率和准确率的影响

- 默认阈值下的 `coverage`

### 5.2 混淆矩阵

建议使用校准后的二分类结果绘制混淆矩阵，体现：

- 真实样本误报情况
- 伪造样本漏报情况

### 5.3 典型样本可视化

建议保留以下样本截图：

1. 正确识别的真实图片
2. 正确识别的伪造图片
3. 被误判为伪造的真实图片
4. 边界困难样本

每张图旁边可附：

- 标签

- `fusion_score`
- 是否触发 `review`

## 6. 如何把这部分写成“工作量”

这部分不是简单“调参数”，而是完整的实验工作，可以从以下角度体现工作量：

1. 构建了外部测试集抽帧数据。
2. 编写了批量评测脚本，支持逐张调用系统 API 输出结果。
3. 编写了离线阈值扫描脚本，对多组真假阈值组合进行系统搜索。
4. 将人工复核率、覆盖率、真实样本准确率、伪造样本准确率等指标纳入综合分析。
5. 基于实验结果定位出系统当前主要瓶颈为跨域泛化不足，而非单纯阈值设置失当。

## 7. 后续与消融实验的衔接方式

阈值调配可以直接作为消融实验中的一项：

1. 启发式方法
2. EfficientNet 视觉模型
3. EfficientNet + XGBoost 融合模型
4. 融合模型 + 阈值校准

其中第 4 项就可以直接使用本轮实验结果。

## 8. 每次实验如何留档，才能方便写论文

每做一轮实验，建议至少保留以下材料：

1. 输入数据范围
   - 哪个数据集
   - 抽了多少图
   - 真假分布如何
2. 运行命令
   - 完整命令行单独保存
3. 运行配置
   - 模型版本
   - 阈值
   - LLM 是否启用
4. 输出结果
   - CSV
   - JSON
   - 终端摘要
5. 样例图片
   - 正确样本
   - 错误样本
   - 边界样本
6. 自己的结论记录
   - 本轮问题是什么
   - 本轮变化是什么
   - 本轮结果是否比上一轮更好

## 10. 当前最适合写进论文的表述重点

你这部分现在最适合强调的不是“已经把效果做到很高”，而是：

1. 通过实验定位出默认阈值是导致高复核率的重要原因。
2. 通过离线阈值校准显著提升了系统自动判定覆盖率。
3. 通过进一步分析发现模型在外部数据集上仍存在真实样本误报偏高问题。
4. 这为后续模型重训与跨域优化提供了实验依据。

这样写，逻辑完整，也能体现你做了实打实的系统分析与优化工作。
